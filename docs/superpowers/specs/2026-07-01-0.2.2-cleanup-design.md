## 3. 技术方案

### 3.1 Web UI slash 命令分发

**现状分析**：
- 后端 slash 命令分发**已实现**：`runInboundLoop` 中的 `SlashCommandHandler` 拦截 `/` 开头消息，走 `CommandRegistry.resolve()` + `command.execute()`
- 前端 chat-page.ts 可能存在拦截逻辑（需检查），或 `/` 开头消息未通过 WebSocket 发送

**方案**：
1. 检查 `src/access/chat-page.ts` 是否有前端代码拦截 `/` 开头消息
2. 若有拦截：移除拦截，让 `/` 开头消息作为普通文本通过 WebSocket 发送
3. 若无拦截：说明 Web UI 已支持，只需浏览器实测验证
4. 若发现问题在后端 SlashCommandHandler 未注入 WebSocket 路径：修复注入逻辑

**验证**：
- 浏览器输入 `/help` 发送，应看到命令输出（Available commands 列表）
- 浏览器输入 `/session` 发送，应看到当前属性列表
- 浏览器输入 `/unknowncmd` 发送，应看到 "❌ unknown command" 错误

**风险**：极低。若前端无拦截逻辑，则 0 代码改动。
