> **状态：🚧 设计草案（部分确认）** — 核心架构已确认，0.3.1/0.3.2 详细范围待讨论。

## 1. 背景与目标

### 1.1 核心诉求

aptbot 作为个人助理，需要支持用户建立多个独立的领域专家 agent：

- 每个 agent 有独立个性（personality）+ 独立技能（skills）+ 独立记忆（memory）
- 每个 agent 下允许有多个对话（sessions），sessions 间共享记忆
- 长期积累：agent 越用越懂用户、越用越专业

**典型场景**：英语学习 agent 长期积累用户的语法易错点、词汇偏好、口语模式。

### 1.2 设计原则

参考 OpenClaw 模式：
- 多 md 文件管理 agent 配置
- 记忆用 Markdown 文件（不用向量数据库）
- 向导模式引导用户谨慎创建 agent

### 1.3 积累类型（4 类）

| 类型 | 含义 | 0.3.0 | 0.3.1+ |
|---|---|---|---|
| 用户偏好积累 | 跨 session 记住用户偏好/习惯 | 手动编辑 MEMORY.md | 半自动（提议-确认） |
| 领域知识积累 | agent 知识库增长 | 手动编辑 MEMORY.md | 半自动 |
| 技能增长 | agent 学会新技能/工具 | 不做 | 0.3.2 |
| 个性演化 | systemPrompt 可调整 | 手动编辑 AGENT.md | 0.3.2 半自动 |

## 3. 核心架构

### 3.1 AgentProfile 实体

每个 agent 是一个目录，包含多个 md 文件：

```
data/users/<userId>/agents/<agentSlug>/
  AGENT.md            # 个性配置（frontmatter + body）
  MEMORY.md           # 共享长期记忆（跨 session，手动编辑）
  sessions/
    <sessionId>.jsonl  # 该 agent 的 session 文件
```

**AGENT.md 格式**：

```markdown
---
name: english-tutor
description: 英语学习助手，专注语法、词汇、口语训练
userId: <userId>
createdAt: 1719700000000
updatedAt: 1719700000000
---

你是英语学习助手，语调亲切，鼓励用户尝试表达。
重点：纠正语法错误时先肯定再修正，提供地道表达建议。
```

- frontmatter：`name` / `description` / `userId` / `createdAt` / `updatedAt`
- body：systemPrompt 正文

### 3.2 agentId 设计

- **slug 格式**（如 `english-tutor`，非 UUID）
- 正则 `/^[a-z0-9-]{3,64}$/`
- 与 sessionId（UUID）分离
- 内置 `default` agent，每个用户首次访问时自动创建

### 3.3 Session ↔ Agent 关系（强约束）

[SessionMetadata](file:///Users/evan/projects/aptbot/src/core/memory/types.ts) 扩展：

```typescript
export interface SessionMetadata {
  readonly id: string;
  readonly createdAt: number;
  readonly updatedAt: number;
  readonly label?: string;
  readonly passedSessions?: number;
  readonly userId?: string;
  readonly agentId: string;  // 新增：必填，归属的 agent
}
```

- 新建 session 必须指定 agentId
- agentId 在 session 创建时确定且不可变
- 现有 sessions 自动迁移到对应用户的 `default` agent

### 3.4 存储路径

| 实体 | 路径 |
|---|---|
| Agent 配置 | `data/users/<userId>/agents/<slug>/AGENT.md` |
| Agent 共享记忆 | `data/users/<userId>/agents/<slug>/MEMORY.md` |
| Session 文件 | `data/users/<userId>/agents/<slug>/sessions/<sessionId>.jsonl` |
| User 存储 | `data/users.jsonl`（不变） |

### 3.5 系统访问约束

[systemPrompt 约束](file:///Users/evan/projects/aptbot/src/server.ts) 更新：
- 禁止访问 `data/users/*/agents/*/sessions/`（原 `data/sessions/`）
- 允许通过 read_agent_memory 工具访问 `data/users/<当前userId>/agents/<当前agentId>/MEMORY.md`
- 禁止访问其他 agent 的 MEMORY.md

## 4. 共享记忆机制

### 4.1 MEMORY.md 内容结构

```markdown
# English Tutor Memory

## User Preferences
- 偏好正式语调
- 常问商务英语

## Domain Knowledge
- 用户当前水平：B2
- 易错点：第三人称单数 -s

## Learned Patterns
- 用户常用 "could you" 而非 "can you"
```

- 分 3 section（对应 0.3.1 半自动积累分类）
- Markdown 纯文本，无 frontmatter
- 软上限：≤ 8KB（超出返回 `memory_too_large`，引导分段读取）

### 4.2 read_agent_memory 工具

```typescript
export interface ReadAgentMemoryParams {
  section?: 'preferences' | 'knowledge' | 'patterns' | 'all';
}

export interface ReadAgentMemoryDetails {
  bytesRead: number;
  truncated: boolean;
}

export const readAgentMemoryTool: AgentTool<ReadAgentMemoryParams, ReadAgentMemoryDetails>;
```

- 读取范围：仅当前 agent 的 MEMORY.md
- 路径约束：工具内部硬编码 `data/users/${currentUserId}/agents/${currentAgentId}/MEMORY.md`，不接受路径参数
- 大小限制：>8KB 返回 `memory_too_large`

### 4.3 write_agent_memory 工具（带确认）

```typescript
export interface WriteAgentMemoryParams {
  section: 'preferences' | 'knowledge' | 'patterns';
  content: string;
  mode: 'append' | 'replace';
}
```

**提议-确认机制**：
1. agent 调用工具
2. 工具不立即写入，返回提议内容
3. CLI `[y/N]` / WebUI 按钮确认
4. 30s 超时视为拒绝
5. 确认后写入（write-to-tmp + rename 原子操作）

### 4.4 上下文注入策略

**0.3.0 不自动注入 MEMORY.md 到 systemPrompt**：

- 自动注入增加每 turn token 消耗
- 让 agent 自主决定何时读取（类似 Skills 系统的 L1 索引设计）

**替代方案**：systemPrompt 末尾追加引导句

```
你可以使用 read_agent_memory 工具读取你的长期记忆（用户偏好/领域知识/学习模式）。
在需要时主动调用，避免每次 turn 都读取。
```

## 5. Agent 切换 UI 与交互

### 5.1 命令系统

| 命令 | 行为 |
|---|---|
| `/agent` | 列出所有 agents（标记当前） |
| `/agent <slug>` | 切换到指定 agent |
| `/agent new <slug>` | 创建新 agent（基础向导引导） |
| `/agent info` | 显示当前 agent 的 AGENT.md 内容 |
| `/agent edit` | 编辑 AGENT.md |

### 5.2 WebUI 侧边栏

```
┌─────────────────────┐
│ aptbot              │
│                     │
│ AGENTS              │
│ ─────────────────── │
│ ▸ english-tutor     │  ← 折叠展开该 agent 的 sessions
│ ▸ code-reviewer     │
│ ▸ default           │
│                     │
│ + 新建 agent        │
│                     │
│ english-tutor ▾     │  ← 当前 agent
│   ─ session 1       │
│   ─ session 2       │
│   + 新会话           │
│                     │
│ user@example.com    │
│ 登出                │
└─────────────────────┘
```

- 顶层：agents 列表（点击切换 agent）
- 二级：当前 agent 的 sessions（折叠展开）

### 5.3 基础向导模式

`/agent new <slug>` 创建流程：
1. 引导填写 name（slug）
2. 引导填写 description
3. 引导填写 personality（systemPrompt body）
4. 生成 AGENT.md
5. 创建空 MEMORY.md
6. 提示用户可以开始对话

### 5.4 HTTP API 扩展

| 端点 | 方法 | 行为 |
|---|---|---|
| `/api/agents` | GET | 列出当前用户的所有 agents |
| `/api/agents/:slug` | GET | 获取 agent 详情 |
| `/api/agents/:slug/sessions` | GET | 列出该 agent 的 sessions |
| `/api/agents/:slug/memory` | GET | 获取 MEMORY.md |
| `/api/agents/:slug/memory` | PUT | 更新 MEMORY.md |

### 5.5 关键交互约束

1. Agent 切换不丢失当前 session（当前 session 保持打开，输入框禁用 + 提示）
2. 新消息自动归属当前 agent
3. session 不能跨 agent 转移（agentId 不可变）

## 6. 迁移策略

### 6.1 现有 sessions 自动迁移

```
0.3.0 升级时：
1. 扫描 data/sessions/*.jsonl
2. 读取每个 session 的 .meta.json 获取 userId
3. 按 userId 分组：
   - 有 userId 的 → data/users/<userId>/agents/default/sessions/<id>.jsonl
   - 无 userId 的 → 为匿名生成 UUID 作为伪 userId，创建 default agent 归属
4. 为每个 userId 创建 default agent 的 AGENT.md（若不存在）
5. 更新 .meta.json 添加 agentId: 'default'
```

### 6.2 default agent 配置

- 每个用户首次访问时自动创建私有 `default` agent
- AGENT.md body 使用当前 systemPrompt（向后兼容）
- 不可删除（但可编辑）

## 7. 错误处理与边界

| 场景 | 处理 |
|---|---|
| AGENT.md 不存在/损坏 | fallback 到 default agent systemPrompt + warn |
| MEMORY.md 不存在 | read_agent_memory 返回空内容（非错误） |
| MEMORY.md > 8KB | 返回 `memory_too_large` |
| write_agent_memory 用户拒绝 | 工具返回 `proposed: false` |
| write_agent_memory 30s 超时 | 视为拒绝 |
| agentId 不存在 | `/agent <slug>` 返回错误 + 列出可用 agents |
| 跨用户访问 agent | HTTP 403 + WebSocket 忽略 |
| 迁移中断 | 幂等设计，重新运行可继续 |

### 7.1 边界约束

- **路径遍历**：agentSlug 严格正则 `/^[a-z0-9-]{3,64}$/`
- **跨用户隔离**：所有 agent 操作校验 `agent.userId === currentUserId`
- **MEMORY.md 写入原子性**：write-to-tmp + rename
- **并发写入**：per-agentId mutex

## 10. 风险与依赖

### 10.1 前置依赖（0.2.2）

0.3.0 多 agent 系统依赖 0.2.2 完成：
- **Skills 系统**（Task 8）：agent 独立技能的基础
- **MixinProvider**（Task 5）：不同 agent 用不同 provider
- **Hook 系统**（Task 7）：注入 agent 个性
- **Channel 抽象**（Task 12）：多 agent UI 路由

### 10.2 风险点

1. **迁移风险**：现有 sessions 迁移可能中断，需幂等设计 + 备份
2. **跨用户隔离**：路径校验必须严格，否则泄露其他用户 agents
3. **token 消耗**：MEMORY.md 读取增加 token，需控制大小与频率
4. **向导模式复杂度**：基础向导看似简单，但交互流程需仔细设计
5. **与 0.2.2 并行风险**：若 0.3.0 部分并行，需明确依赖边界
