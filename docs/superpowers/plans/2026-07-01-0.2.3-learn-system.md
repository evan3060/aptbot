# aptbot 0.2.3 学习系统实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 aptbot 网站首页新增知识栏目，展示 19 篇结构化文章（Track 1: Agent 体系实践 13 篇 + Track 2: AI 辅助编码实践 6 篇），并增加用户反馈区，访问者可提交想法/需求记录到待办。

**Architecture:** 新增 src/learn/ 模块承载 markdown 文章与 ArticleLoader（gray-matter 解析 frontmatter + marked 运行时渲染 + mtimeNs 热重载）；src/access/learn-page.ts 承载 HTML 生成器（列表页/文章页/反馈页）；src/access/feedback-api.ts + src/infrastructure/feedback-storage.ts 承载反馈 API 与 JSONL 存储；扩展现有 websocket-server.ts 路由分发与 landing-page.ts 第 6 section。

**Tech Stack:** TypeScript (ESM) / Node.js / zod (schema 校验) / gray-matter (frontmatter 解析) / marked (markdown 渲染) / vitest (测试) / Ink (CLI) / 现有 adept tokens CSS 变量 + Inter 字体

## Global Constraints

- 文章存储：markdown 源文件 + frontmatter，放在 src/learn/articles/ 目录
- 渲染方式：marked 运行时渲染 + 内存缓存，无构建步骤
- 配置默认值：learnPage 未设置 → false（opt-in）；feedbackEnabled 未设置 → true（默认启用）
- learnEnabled 定义：landingPage === true && learnPage === true
- 路由分发顺序：/api/feedback 优先于 /api/* > / 与 /demo > /learn > /learn/:slug > /feedback > 404 兜底
- 所有 HTML 响应头含 Content-Type: text/html; charset=utf-8 + Cache-Control: no-cache, no-store, must-revalidate + X-Content-Type-Options: nosniff
- 不含 emoji（用 CSS 几何符号/文字标签替代，如 planned 用 "PLANNED" 文字标签）
- 文章信任边界内，不引入 DOMPurify
- 不引入语法高亮（marked 输出纯 pre/code）
- 不做 E2E 浏览器自动化测试
- 不动 src/webui/ Lit 实现、不动 src/cli/ 现有 Ink 组件（仅新增 /feedback 命令）、不动 /api/* 现有路由
- 描述性内容用中文，参数/变量/类型名/文件路径/命令/commit message 用英文
- plan 不含具体代码实现，代码在 TDD 阶段根据测试错误驱动编写
- 每篇 markdown 文章 1500-2500 字，概念性理解为主，必要时少量伪代码（≤20 行），概念清晰则无代码
- Track 1 与 Track 2 完全分离，编号独立（Track 1: 01-13，Track 2: 01-06），slug 全局唯一
- Track 2 文章与具体项目无关，案例不深入 aptbot 细节

- lastUpdated: 正则 ^\d{4}-\d{2}-\d{2}$

Article 接口：含 meta (ArticleMeta) + renderedHtml (string | null，planned 为 null) + markdownBody (string)

ArticleState 接口：含 articles (readonly Article[]) + tracks (readonly TrackMeta[]) + bySlug (ReadonlyMap<string, Article>) + byTrack (ReadonlyMap<string, readonly Article[]>)

ArticleNav 接口：含 prev (Article | null) + next (Article | null)

TrackMeta 接口：含 id / title / description / order

TRACKS 常量：含两项 — agent-practice (Agent 体系实践) + ai-coding-practice (AI 辅助编码实践)

## Task 2: article-loader.ts — ArticleLoader 类

**Files:**
- Create: src/learn/article-loader.ts
- Test: tests/learn/article-loader.spec.ts
- Fixtures: tests/learn/fixtures/（合法/非法 frontmatter markdown 文件）

**Interfaces:**
- Consumes: ArticleMeta / Article / ArticleState / ArticleNav / TRACKS（from Task 1）/ marked / gray-matter / zod
- Produces: ArticleLoader 类（构造函数接收 articlesDir 参数 + 可选 logger）

**行为描述：**

ArticleLoader 类方法：
- load(): Promise<void> — 扫描 articlesDir 下 *.md（fs.readdirSync 按文件名排序），逐文件 gray-matter 解析 → zod safeParse 校验 → marked 渲染（published）/ 跳过（planned）→ 唯一性校验 → 排序分组 → 缓存 ArticleState
- getState(): ArticleState — 返回当前缓存快照（未 load 时抛错或返回空状态）
- getBySlug(slug): Article | null — 从 bySlug Map 查找
- getArticleNav(slug): ArticleNav — 返回同 track 内上一篇/下一篇（按 order 排序，跨 chapter 边界）
- 热重载：每次 getState/getBySlug/getArticleNav 调用时检查 articlesDir mtimeNs + 每个文件 mtimeNs 数组快照，变化则触发 reload（per-loader mutex 串行化，并发请求等同一 reload 完成）

加载流程错误处理（warning + 跳过，不阻塞启动）：
- gray-matter 解析失败：stderr warning（含文件名）+ 跳过
- zod 校验失败：stderr warning（含文件名 + zod 错误详情）+ 跳过
- slug 重复：warning + 保留 order 较小者
- prerequisites 引用不存在 slug：warning + 清空该字段（不阻塞）
- track 不在 TRACKS 注册表：warning + 跳过该文件
- order 全局重复：warning + 文件名兜底排序
- published 但 markdownBody trim 后 < 100 字符：warning（疑似未写完）但不阻塞

marked 渲染配置：gfm true / breaks false。自定义 renderer 为 h2/h3 生成 id（slugify：转小写 → 非字母数字替换为 - → 连续 - 压缩 → 去首尾 -，冲突追加 -2/-3）。为 pre 加 data-language 属性。marked@15 renderer API 为 token-based，实施时查文档确认 API。

边界情况：目录不存在或为空 → 返回空 ArticleState + warning；全部 planned → 不渲染正文；热重载期间请求到达 → mutex 串行化；文件编码 UTF-8（gray-matter 自动剥离 BOM）。

## Task 3: feedback-storage.ts — FeedbackStorage 类

**Files:**
- Create: src/infrastructure/feedback-storage.ts
- Test: tests/infrastructure/feedback-storage.spec.ts

**Interfaces:**
- Consumes: 现有 jsonl.ts（append-only 写入）/ jsonl-mutex.ts（per-file async-mutex）
- Produces: FeedbackStorage 类 + FeedbackEntry 接口 + FeedbackInput 接口 + FeedbackListFilter 接口

**行为描述：**

FeedbackEntry 结构：id (fb-<timestamp>-<rand6>) / message / category (general | article | bug | feature) / articleSlug? / contact? / ip / userAgent? / status (open | resolved | archived) / note? / createdAt (ISO 8601) / moderatedAt?

FeedbackStorage 方法：
- append(input: FeedbackInput): Promise<FeedbackEntry> — 自动生成 id（fb-<Date.now()>-<rand6hex>）+ createdAt（new Date().toISOString()），用 per-file mutex 独占锁 append 一行 JSON 到 <dataDir>/feedback.jsonl，返回完整 entry
- list(filter: FeedbackListFilter): Promise<{ items: FeedbackEntry[]; total: number }> — 增量流式解析 JSONL，支持 status 过滤（默认 open）/ category 过滤 / limit（1-100 默认 50）+ offset（≥0 默认 0）分页，total 为过滤后总数
- moderate(id: string, update: { status: 'resolved' | 'archived'; note?: string }): Promise<FeedbackEntry | null> — findById 查找，不存在返回 null；存在则用 per-file 锁重写整个 JSONL 文件（更新该条 status/note/moderatedAt），返回更新后的 entry
- findById(id: string): Promise<FeedbackEntry | null> — 增量流式解析查找

并发安全：append 用 per-file mutex 独占锁；list/findById 读取无需锁（增量流式解析容忍并发 append）；moderate 重写整个文件用 per-file 锁。

损坏修复：复用现有 JSONL 增量流式解析 + 破损行容错（skip + stderr warning，不 crash）。不引入 fs.truncateSync 自动修复（反馈数据丢失可接受）。

文件不存在时 append 自动创建（fs.appendFile 自动创建）。

## Task 8: feedback-api.ts — handleFeedbackApi 与限流器

**Files:**
- Create: src/access/feedback-api.ts
- Test: tests/access/feedback-api.spec.ts

**Interfaces:**
- Consumes: FeedbackStorage（from Task 3）/ ArticleLoader（from Task 2，校验 articleSlug）/ 现有 authToken 机制
- Produces: handleFeedbackApi(req, res, pathname, feedbackStorage, articleLoader?, authToken?) 函数 + RateLimiter 类（per IP 10/min + 60/hour 滑动窗口，内存 Map）

**行为描述：**

handleFeedbackApi 处理三类请求：

POST /api/feedback（提交反馈）：
- 解析 JSON body
- zod 校验：message (string min 1 max 2000 trim 后非空) / category (enum general | article | bug | feature，默认 general) / articleSlug (仅 category=article 时必填，正则 ^[a-z0-9-]+$，且 articleLoader.getBySlug 校验存在) / contact (string max 120 可选)
- 限流检查（per IP 10/min + 60/hour，超限返回 429 + Retry-After header）
- 校验失败返回 400 + { ok: false, error, field }
- 校验通过调用 feedbackStorage.append，返回 200 + { ok: true, id }

GET /api/feedback（列出反馈，需 auth）：
- 校验 Authorization: Bearer <token>，不匹配返回 401
- 解析 query：limit (1-100 默认 50) / offset (≥0 默认 0) / category (可选) / status (open | resolved | archived，默认 open)
- 调用 feedbackStorage.list，返回 200 + { ok: true, items, total }

POST /api/feedback/:id/moderate（审核，需 auth）：
- 校验 auth token，不匹配返回 401
- 解析 pathname 提取 id
- 解析 body：status (resolved | archived) / note (max 500 可选)
- 调用 feedbackStorage.moderate，不存在返回 404，存在返回 200 + { ok: true, entry }

RateLimiter 类：per IP 内存 Map 滑动窗口。checkRateLimit(ip) 返回 { allowed: boolean, retryAfter: number }。重启重置（可接受）。
