## 范围 3：新功能逐项验证（0.2.2 十项新功能）

### 3.1 Task 1: per-sessionKey ring buffer 分片 + LRU

- [x] 单 sessionKey 长对话（>1000 条消息）不导致内存膨胀（ring buffer 截断到 1000）（websocket-history-replay.spec.ts 覆盖）
- [x] 多 sessionKey 场景下全局内存上限 50000 触发 LRU 淘汰最旧 sessionKey（E2E + 单测覆盖）
- [x] 被淘汰的 sessionKey 重新活跃时可正常写入新 buffer（单测覆盖）

### 3.2 Task 2: turn_busy 响应

- [x] 同一 sessionKey 快速连续发送 2 条消息，第二条前端显示"等待中... (前方 1 条消息)"（inbound-serialization.spec.ts 覆盖）
- [x] 第一条 turn 结束后，第二条开始处理，等待提示消失（E2E 覆盖）
- [x] 不同 sessionKey 并发消息互不影响（无 turn_busy）（E2E 覆盖）

### 3.3 Task 3: JSONL 历史持久化

- [x] 重启服务器后，刷新页面，旧会话历史正常回放（最近 20 条 message）（websocket-history-replay.spec.ts 覆盖）

### 3.4 Task 4: HttpOnly cookie 安全增强
