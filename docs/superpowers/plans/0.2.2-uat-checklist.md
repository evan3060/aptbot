## 范围 3：新功能逐项验证（0.2.2 十项新功能）

### 3.1 Task 1: per-sessionKey ring buffer 分片 + LRU

- [ ] 单 sessionKey 长对话（>1000 条消息）不导致内存膨胀（ring buffer 截断到 1000）
- [ ] 多 sessionKey 场景下全局内存上限 50000 触发 LRU 淘汰最旧 sessionKey
- [ ] 被淘汰的 sessionKey 重新活跃时可正常写入新 buffer

### 3.2 Task 2: turn_busy 响应

- [ ] 同一 sessionKey 快速连续发送 2 条消息，第二条前端显示"等待中... (前方 1 条消息)"
- [ ] 第一条 turn 结束后，第二条开始处理，等待提示消失
- [ ] 不同 sessionKey 并发消息互不影响（无 turn_busy）

### 3.3 Task 3: JSONL 历史持久化

- [ ] 重启服务器后，刷新页面，旧会话历史正常回放（最近 20 条 message）
- [ ] 回放的消息带 `replay: true` 标记，前端不重复渲染
- [ ] 回放仅含 message 类型，不含 tool_call（不泄漏内部状态）
- [ ] JSONL 文件损坏时自动截断修复（服务重启不崩溃）

### 3.4 Task 4: HttpOnly cookie 安全增强

- [ ] 登录成功后浏览器 DevTools → Application → Cookies 可见 token cookie
- [ ] Cookie 属性：`HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=2592000`（HTTP localhost 下 Secure 条件性省略）
- [ ] 刷新页面后登录态保持（cookie 自动携带）
- [ ] WebSocket 连接使用 cookie 鉴权（无需 URL ?token=）
- [ ] 退出登录后 cookie 清除

### 3.5 Task 5: MixinProvider 多 provider 故障转移

> **注意：** 需配置多 provider 才能验证。若当前配置仅单 provider，标记为"⏸ 需多 provider 配置"。

- [ ] 配置主 provider + 备 provider，主 provider 故障时自动 fallback 到备 provider
- [ ] 主 provider 恢复后 5 分钟（springBackMs）自动弹回
- [ ] 所有 provider 故障时显示 AggregateError 错误信息

### 3.6 Task 6: Config 热重载

- [ ] 修改 `config/aptbot.json` 的 model 配置，保存
- [ ] 下一个 turn 自动使用新 model（无需重启服务器）
- [ ] 当前进行中的 turn 不受影响（用旧配置完成）
- [ ] 配置非法时服务器日志打印 warn，继续用旧配置

### 3.7 Task 7: Hook 系统（8 hook 点）

> **注意：** Hook 系统为基础设施，需编写 hook 插件才能验证。若未配置 hook，标记为"⏸ 需 hook 插件"。

- [ ] 在 `~/.aptbot/hooks/` 或 `.agents/hooks/` 放置测试 hook，8 个 hook 点均被触发
- [ ] hook 抛错时不影响主流程（吞异常 + stderr 打印）
- [ ] priority 升序排序生效（低 priority 先执行）

### 3.8 Task 8 + 9: Skills 系统 + L1 索引

- [ ] 在 `~/.aptbot/skills/` 创建测试 skill（含 SKILL.md + frontmatter）
- [ ] agent system prompt 包含 L1 索引（name/description/行数/字节/tags）
- [ ] agent 调用 read_file 读取 SKILL.md 后，该 skill 的 lastUsed 更新（下次 L1 索引排序提前）
- [ ] 创建 >4K token 的多个 skill，L1 索引截断为"前 N 个完整 + 名字列表"
- [ ] Config 热重载时 Skills 也重载（新增/修改/删除 skill 生效）

### 3.9 Task 10: Session 自动摘要命名

- [ ] 新会话发送首条消息后，turn_end 触发异步摘要生成
- [ ] 侧边栏该会话 label 显示为 LLM 生成的摘要（≤20 字符）
- [ ] 继续对话后摘要可能刷新（无 custom label 时）
- [ ] 手动 `/label <名称>` 后，该会话 label 固定为用户设置，后续 turn 不再自动摘要
- [ ] LLM 摘要失败时不报错，保留默认 label（首条消息 preview）

### 3.10 Task 11: /session 动态属性

- [ ] `/session` 无参数列出所有属性当前值
- [ ] `/session temperature 0.7` 设置成功，回复确认
- [ ] `/session temperature` 返回当前值 0.7
- [ ] `/session temperature 5` 返回错误 + 列出合法范围 `[0, 2]`
- [ ] `/session reasoningEffort invalid` 返回错误 + 列出 6 个合法值
- [ ] `/session.reset` 重置所有属性
- [ ] 设置属性后（若用 MixinProvider）广播到所有子 provider

### 3.11 Task 12: Channel 接口抽象

> **注意：** Channel 抽象为 0.4.0 IM 铺路，MVP 阶段仅 WebSocket 实现。此项主要验证不回归。

- [ ] WebSocket 仍作为 Channel 实现正常工作（聊天/工具/session 均正常）
- [ ] 多客户端（多浏览器标签）连同一 sessionKey，消息广播到所有客户端
- [ ] 客户端断开后自动 unbind（不再收到该 session 的消息）
