## Task 1: Web UI slash 命令分发修复

**价值：** 让 Web UI 支持 `/help` `/session` `/new` 等 slash 命令，复用后端已实现的 CommandRegistry。

**文件：**
- Check: `src/access/chat-page.ts`（检查是否有前端拦截 `/` 开头消息的逻辑）
- Modify（若有拦截）: `src/access/chat-page.ts`（移除拦截，让 `/` 开头消息走正常 WebSocket 发送）
- Modify（若后端未注入）: `src/server.ts` 或 `src/access/websocket-server.ts`（确保 SlashCommandHandler 注入 WebSocket 路径）
- Test: `tests/access/websocket-slash-command.spec.ts`（新建，若代码有改动）

**设计契约：** [cleanup-design.md §3.1](../specs/2026-07-01-0.2.2-cleanup-design.md#31-web-ui-slash-命令分发)

**行为：**
1. 检查 chat-page.ts 中是否有前端代码拦截 `/` 开头消息（如 `if (text.startsWith('/')) { ... return; }` 之类的逻辑）
2. 若有拦截：移除拦截，让 `/` 开头消息作为普通文本通过 WebSocket 发送
3. 若无拦截：检查后端 websocket-server.ts 是否将 SlashCommandHandler 注入 runInboundLoop
4. 若后端未注入：修复注入逻辑，确保 WebSocket 路径的 runInboundLoop 有 slashHandler 参数
5. 若两者都正常：说明 Web UI 已支持，只需浏览器实测验证

**验证：**
- 浏览器输入 `/help` 发送，应看到命令输出（Available commands 列表）
- 浏览器输入 `/session` 发送，应看到当前属性列表
- 浏览器输入 `/unknowncmd` 发送，应看到错误提示
- 若代码有改动：`npm run test -- tests/access/websocket-slash-command.spec.ts` 全绿
- 全量回归：`npm test` 保持 938+ 全绿

**Commit：** `fix: enable slash command dispatch in Web UI`（若有代码改动）或无 commit（若 0 代码改动）

## Task 2: Hook/Skill 示例插件 + 文档

**价值：** 提供可测试的示例插件验证 Hook/Skill 基础设施工作正常，文档指导用户创建自定义插件。

**文件：**
- Create（本地，不进 git）: `~/.aptbot/hooks/01-log-turn.js`（Hook 示例，CommonJS 模块，turn_before/after 打印 stderr）
- Create（本地，不进 git）: `~/.aptbot/skills/example/SKILL.md`（Skill 示例，含 frontmatter + body）
- Create: `docs/hooks-guide.md`（Hook 8 点说明 + ctx 结构 + priority + 两层目录 + 示例）
- Create: `docs/skills-guide.md`（frontmatter 规范 + 校验规则 + 两层加载 + L1 索引 + 示例）

**设计契约：** [cleanup-design.md §3.2](../specs/2026-07-01-0.2.2-cleanup-design.md#32-hookskill-示例插件)

**行为：**

**Hook 示例文件**（`~/.aptbot/hooks/01-log-turn.js`）：
- CommonJS 模块，`module.exports = { priority, turn_before, turn_after }`
- priority: 10（低优先级先执行）
- turn_before: `process.stderr.write('[hook] turn starting\n')`
- turn_after: `process.stderr.write('[hook] turn ended\n')`
- 不抛错（避免影响主流程）

**Skill 示例文件**（`~/.aptbot/skills/example/SKILL.md`）：
- frontmatter: `name: example` / `description: An example skill for testing` / `disableModelInvocation: false`
- body: 简短说明示例 skill 用途

**文档 docs/hooks-guide.md：**
- 8 个 hook 点说明：agent_before/after, turn_before/after, llm_before/after, tool_before/after
- ctx 结构说明（可 mutate，链式传递）
- priority 升序排序规则
- 异常吞掉策略（hook 抛错不影响主流程）
- 两层目录加载（`~/.aptbot/hooks/` + `.agents/hooks/`，workspace 覆盖 builtin）
- 示例 hook 文件代码

**文档 docs/skills-guide.md：**
- frontmatter 规范：name（a-z0-9-, ≤64 字符）/ description（≤1024 字符）/ disableModelInvocation
- 校验规则与 SkillDiagnostic warning
- 两层加载（`~/.aptbot/skills/` + `src/skills/`，workspace 覆盖 builtin 同名）
- L1 索引字段：contentLines / contentBytes / tags / lastUsed
- 4K token 预算截断策略
- read_file 特判更新 lastUsed 机制
- 示例 SKILL.md 文件

**验证：**
- 启动服务器，发送一条消息，检查 stderr 日志含 `[hook] turn starting` 和 `[hook] turn ended`
- 检查 agent system prompt 含 `example` skill 的 L1 索引（name/description/行数/字节）
- `read_file` 读取 `~/.aptbot/skills/example/SKILL.md` 后，重启服务器，检查 example 在 L1 索引排序提前（lastUsed 更新）
- 文档无测试（纯文档）

**Commit：** `docs: add hooks and skills guide with example plugins`
